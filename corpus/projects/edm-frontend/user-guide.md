# User Guide

本文件寫給**第一次使用 EDM 系統**的行銷人員與業務看。不談 JWT、不談架構,只講你會看到什麼、應該按哪裡。

---

## 1. 這是什麼?

EDM 是一套**內部行銷活動管理工具**,讓你可以:

- 管理人員名單(會員 / 客戶)與群組
- 建立活動、設計邀請信、寄給選定群組
- 為活動產製 Google 報名表、追蹤回應狀況
- 查看活動成效(開信率、報名轉換率)

**你不需要記密碼** — 系統登入是透過中台 SSO,輸入 Email 收信點連結即可。

---

## 2. 第一次登入

### 2.1 進入系統

打開瀏覽器,輸入系統網址(由內部公告或 IT 提供):

```
http://edm.example.com/      ← Production
EdmFront/         ← 本機開發
```

### 2.2 自動跳轉到中台登入

第一次進站,系統會自動把你帶到**中台登入頁**:

```
EDM (你看不到內容)
      ↓
中台登入頁 ─── 輸入 Email
      ↓
信箱收到「Middle Platform 登入連結」信
      ↓
點信中的「登入」按鈕(連結 15 分鐘有效)
      ↓
返回 EDM 看到主畫面
```

> 若收不到信:檢查垃圾信夾;若仍無,聯絡 IT 確認你的 Email 是否在系統內。

### 2.3 進入後看到的畫面

成功登入後會看到 **EDM 首頁**:

- 頂部:你的姓名 / 登出按鈕
- 左側:側邊選單(群組管理 / 人員管理 / 活動管理 / SA 文件)
- 主區域:預設儀表板或第一個有權限的頁

---

## 3. 群組管理

### 3.1 看群組列表

側邊選單 → **群組管理 → 列表**

你會看到:

- 群組名稱、備註、建立者、目前狀態(啟用 / 停用)
- 每筆有「詳細」按鈕

### 3.2 新增群組

列表頁右上角 → **新增群組**

填寫:

- **群組名稱**(必填,如「2026 年五月發表會邀請對象」)
- **備註**(選填)

確認後立即建立,可以馬上去**人員管理**把人加進來。

### 3.3 啟用 / 停用群組

每筆右側有一個**開關 (ElSwitch)**,直接點即可切換。停用後該群組**不會出現在活動的「邀請對象」選單裡**,但歷史活動的關聯不受影響。

### 3.4 看群組詳細

點某筆右側的「詳細」進入,有三個分頁:

1. **人員列表** — 這個群組裡有哪些人
2. **活動列表** — 這個群組曾被邀請過的活動
3. **分析報告**(開發中) — 這個群組的歷史開信率、報名率

## 4. 人員管理

### 4.1 Excel 批次匯入(常用)

側邊選單 → **人員管理** → **Excel 匯入**

操作步驟:

1. **選目標群組**(這一步是**鎖定的**,匯入後人員會自動加進此群組,避免你忘記選)
2. 點**選擇檔案**,挑你準備好的 Excel
3. 系統解析後會**預覽前 10 列**,確認欄位對應正確
4. 沒問題就**確認匯入**
5. 完成後顯示「成功 X 筆 / 失敗 Y 筆」(失敗有原因列表,通常是 email 格式錯)

> Excel 範本可在匯入頁右上角下載。**重要欄位**:姓名、Email、Mobile(可多筆,用半形逗號分隔)、組織。

### 4.2 編輯個別人員資料

人員列表 → 點某筆 → 進入「人員詳細」

可以編輯:

- **姓名 / 狀態**
- **Email**(可加多筆,標一筆為主要)
- **Mobile**(可加多筆)
- **業務歸屬**(`sales_email`,業務用)

> Email 與 Mobile 是**獨立 endpoint** 編輯的(改完即時 save,不用按「全部儲存」),所以改完看到 toast 就 OK。

### 4.3 業務人員特殊功能

如果你是業務,你會看到頂部多一個過濾器:**只看名下客戶**(預設開啟)。

關掉就能看全部人員,但**只能編輯名下的**。

## 5. 活動管理

### 5.1 建立新活動

側邊選單 → **活動管理** → **新增活動**

需要填的欄位:

| 欄位                | 說明                                       | 必填     |
| ------------------- | ------------------------------------------ | -------- |
| 活動名稱            | 對外顯示的標題                             | ✅       |
| 活動簡介            | 一段話說明                                 | —        |
| 活動內容            | **CKEditor 富文字編輯器**,可貼圖、設定樣式 | ✅       |
| 活動圖片            | 上傳一張(會出現在邀請信頭)                 | ✅       |
| 開始時間 / 結束時間 | datetime 選擇器                            | ✅       |
| 地址 / 地標         | 文字                                       | 地址必填 |
| 活動類別            | 下拉選單                                   | ✅       |

建立後會自動產生**活動單號 `Bxxx`**(系統流水號,不能改)。

### 5.2 維護邀請名單

進活動詳細 → **邀請名單**分頁

兩種方式加入邀請對象:

1. **從群組匯入**(快速):選一個或多個群組,該群組所有人**一次加入**邀請名單(會自動 snapshot 當下的 email / mobile / organization)
2. **手動加單一人員**:從人員列表搜尋後加入

> **同一個人不會重複加**(系統自動去重)。

### 5.3 寄發邀請信

進活動詳細 → **邀請信**分頁

操作:

1. 用 CKEditor 編輯信件內文(可用變數 `{name}`、`{event_title}` 等,系統會自動替換)
2. 點**預覽**確認排版
3. 點**寄出**

系統會**立即回應「已加入隊列」**,實際寄送由背景作業執行(可能幾分鐘到幾十分鐘,看人數)。

> ⚠ **不要重複按「寄出」** — 雖然系統會去重,但連按會浪費 API 呼叫。

### 5.4 為活動建 Google 報名表

進活動詳細 → **報名表**分頁 → **建立 Google 表單**

系統會代你建一張 Google Form(綁定到此活動),回傳:

- 表單 URL(可放進邀請信內文,或直接給人填)
- QR Code(印在實體文宣)

填寫後的回應**每小時自動同步**進系統,你可以在「報名管理」分頁看到並做**審核**(通過 / 不通過)。

## 6. 查看數據(開發中)

側邊選單 → **數據分析**(若你有權限)

可以看:

- **開信率**:寄了多少封、開了多少封
- **點擊率**:信內連結被點了幾次
- **報名轉換率**:多少受邀者最後填了報名表

> ⚠ 目前(2026 Q2)此功能仍在開發,實際數據可能不準。

## 7. SA 文件(系統內查看)

側邊選單最下方 → **SA 文件**

進去可以看:

- 使用案例圖

- ER 圖

這些是給開發者 / SA 看的,**End User 不看也沒關係**。內容跟 GitHub 上的 [`sa-docs/`](.) 同步,正本在 GitHub。

## 8. 常見問題

### Q1: 我登入後過一陣子又被踢出去要重登?

A: 系統 Token 預設 30 分鐘有效。30 分鐘內沒操作就會過期,系統會**自動帶你回中台重新登入**。對應功能在 [user-flow.md 第 4 節](./user-flow.md#4-token-失效流程)。

### Q2: 寄信後多久能收到?

A: 視寄送筆數,通常**幾分鐘到 30 分鐘**。系統使用 AWS SES,會分批發送避免被視為垃圾信。

### Q3: 我手滑刪錯一個群組 / 活動,救得回來嗎?

A: **救得回來!** 系統採 Soft Delete,刪除只是標記,實際資料還在。請聯絡 IT,可從後台 restore。詳見 [edm_backend ADR-0003](../../edm_backend/docs/adr/0003-soft-deletes.md)。

### Q4: 我可以同時開兩個分頁操作嗎?

A: **可以**,但**避免在兩個分頁同時編輯同一筆**(會以後存的為準,前一個的修改會被覆蓋)。

### Q5: 為什麼有時候上傳 Excel 會失敗?

A: 常見原因:

- Excel 格式不對(請用提供的範本)
- 某些列 Email 格式錯誤
- 檔案太大(>10MB 建議分批)

失敗時系統會顯示哪幾列錯,改完重新匯入即可。

### Q6: 為什麼我登入時跳到一個叫「中台 / Middle Platform」的東西?

A: 那是**集中式登入服務**(SSO),整個公司所有內部系統共用同一個登入。好處:

- 一組 Email 登所有系統,不必記多個密碼
- 離職時 IT 一次關掉就所有系統都進不去

詳見中台的 [overview.md](../../Middle_Platform/docs/overview.md)。

| 問題類型                  | 找誰                   |
| ------------------------- | ---------------------- |

| 業務問題(活動規劃、文案)  | 行銷主管               |
